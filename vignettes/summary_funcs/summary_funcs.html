Introduction

The multivar package provides three main functions for summarizing and visualizing fitted models:

  1. summary_multivar() - Comprehensive model summary
  2. print_dynamics() - Visualize dynamics matrices
  3. print_edge_prevalence() - Show edge prevalence across subjects (K>1)

This vignette demonstrates how to use these functions to explore and interpret your multivar model results.

Getting Started

First, let’s load the package and fit a simple model:

library(multivar)
# Simulate data with K=2 subjects
set.seed(123)
sim <- multivar_sim(
  k = 2,
  d = 4,
  n = 100,
  prop_fill_com = 0.3,
  prop_fill_ind = 0.15,
  sigma = diag(4),
  lb = 0.1,
  ub = 0.9
)

# Fit the model
object <- constructModel(sim$data, intercept = FALSE)
#> Warning in constructModel(sim$data, intercept = FALSE): multivar WARNING:
#> common_tvp_effects = TRUE has no effect when tvp = FALSE.
fit <- cv.multivar(object)

Model Summary: summary_multivar()

The summary_multivar() function provides a comprehensive overview of your fitted model.

What Information is Displayed?

Dataset Information

  • Number of datasets (subjects): The K value
  • Number of timepoints: Handles varying lengths across subjects
  • Number of variables: The d value

Model Specification

  • Time-varying parameters: Whether the model includes TVP
  • Period ranges: For TVP models, shows time ranges for each period
  • Intercepts: Whether intercepts are included and penalized

Adaptive Weights

  • Method: LASSO, RIDGE, etc.
  • Type: Adaptive or Standard
  • Lambda selection: Minimum CV error or 1 SE rule
  • Cross-validation: Type and number of folds

Overview of Dynamics

  • Edge counts: Number of non-zero entries
  • Sparsity: Percentage of zero entries
  • Value ranges: Range of non-zero coefficients

Visualizing Dynamics: print_dynamics()

The print_dynamics() function displays dynamics matrices with clear formatting and time subscripts.

Direct Use with Fit Objects

One of the key features is that you can pass the fit object directly:

print_dynamics(fit, digits = 3)

#> V4 .      0.278 .     -0.008

#>

#> ====================================================================== 
#> 
#>    V1    V2    V3    V4   
#> V1 0.044 0.071 0.242 0.496
#> V2 0.638 .     .     .    
#> V3 0.236 .     .     .    
#> V4 .     0.278 .     0.696
#> 
#> Non-zero entries: 8 / 16 (50.0% sparse)

Key Features

  • Time subscripts: Rows labeled as V1_t, columns as V1_{t-1}
  • Dots for zeros: Uses . to display zero entries for better readability
  • Summary statistics: Shows sparsity and value ranges
  • Automatic routing: Detects TVP models and formats accordingly

Customization Options

# More decimal places
print_dynamics(fit, digits = 4)

# Show zeros as "0" instead of "."
print_dynamics(fit, zero_char = "0")

# Without time subscripts
print_dynamics(fit, time_labels = FALSE)

Time-Varying Models

Let’s demonstrate the summary functions with a time-varying parameter (TVP) model.

Fitting a TVP Model

# Create time-varying dynamics
phi_period1 <- matrix(c(
  0.3,  0.4,  0.0,  0.0,
  0.0,  0.5,  0.3,  0.0,
  0.4,  0.0,  0.2,  0.0,
  0.0,  0.3,  0.0,  0.4
), nrow = 4, byrow = TRUE)

phi_period2 <- matrix(c(
  0.4,  0.3,  0.0,  0.2,
  0.0,  0.6,  0.4,  0.0,
  0.3,  0.0,  0.3,  0.0,
  0.0,  0.4,  0.0,  0.5
), nrow = 4, byrow = TRUE)

phi_period3 <- matrix(c(
  0.5,  0.2,  0.0,  0.0,
  0.0,  0.4,  0.5,  0.0,
  0.5,  0.0,  0.4,  0.2,
  0.0,  0.5,  0.0,  0.3
), nrow = 4, byrow = TRUE)

# Generate data
phi_list <- c(
  rep(list(phi_period1), 80),
  rep(list(phi_period2), 80),
  rep(list(phi_period3), 80)
)

data1 <- multivar:::var_sim_growth(
  n = 240,
  phi = phi_list,
  sigma = diag(4),
  intercept = rep(0, 4)
)

# Fit TVP model
object_tvp <- constructModel(
  list(data1),
  intercept = FALSE,
  tvp = TRUE,
  breaks = list(c(80, 160)),
  cv = "blocked",
  nfolds = 5
)

fit_tvp <- cv.multivar(object_tvp)

Edge Prevalence Across Subjects

For models with multiple subjects (K>1), print_edge_prevalence() shows which edges are present across subjects.

Proportion Display

Shows what proportion of subjects have each edge:

print_edge_prevalence(fit, type = "proportion", digits = 2)
#> 
#> ====================================================================== 
#> Edge Prevalence (Proportion of Subjects)
#> ====================================================================== 
#> 
#>    V1   V2   V3   V4  
#> V1 1.00 1.00 1.00 1.00
#> V2 1.00 .    .    .   
#> V3 1.00 .    .    0.50
#> V4 .    1.00 .    1.00
#> 
#>   Universal edges (100%):        8
#>   Common edges (>50%):           8
#>   Rare edges (<50%):             0
#>   Absent edges (0%):             7
#> ======================================================================

Count Display

Shows the actual count of subjects with each edge:

print_edge_prevalence(fit, type = "count")
#> 
#> ====================================================================== 
#> Edge Prevalence (Count of Subjects)
#> ====================================================================== 
#> 
#>    V1  V2  V3  V4 
#> V1 2/2 2/2 2/2 2/2
#> V2 2/2 .   .   .  
#> V3 2/2 .   .   1/2
#> V4 .   2/2 .   2/2
#> 
#>   Universal edges (all 2):       8
#>   Variable edges (between 1-1):  1
#>   Absent edges (0):               7
#> ======================================================================

Interpreting the Output

The summary statistics tell you:

  • Universal edges (100%): Present in all subjects
  • Common edges (>50%): Present in more than half of subjects
  • Rare edges (<50%): Present in less than half of subjects
  • Absent edges (0%): Not present in any subject

This helps identify: - Shared network structure: Universal and common edges represent the common dynamics - Individual variation: Rare edges show subject-specific patterns - Network sparsity: Absent edges indicate where no relationships exist

Complete Workflow Example

Here’s a typical workflow combining all three functions:

# 1. Fit your model
fit <- cv.multivar(object)

# 2. Get comprehensive summary
summary_multivar(fit)

# 3. Visualize dynamics
print_dynamics(fit)

# 4. For K>1: Check edge prevalence
print_edge_prevalence(fit, type = "proportion")

Tips and Best Practices

When to Use Each Function

  • summary_multivar(): Always use this first to understand your model structure
  • print_dynamics(): Use to examine specific dynamics matrices with clear formatting
  • print_edge_prevalence(): Use for K>1 models to understand shared vs. unique structure

Customization

All functions support customization:

# Summary (no customization needed - automatic)
summary_multivar(fit)

# Dynamics with custom formatting
print_dynamics(
  fit,
  digits = 4,              # More precision
  zero_char = "—",         # Custom zero character
  period_labels = c(...)   # For TVP models
)

# Edge prevalence with custom display
print_edge_prevalence(
  fit,
  type = "count",          # Show counts instead of proportions
  digits = 3,              # Decimal places for proportions
  zero_char = "—"          # Custom zero character
)

Working with Different Model Types

K=1 Models

  • summary_multivar() shows single-subject statistics
  • print_dynamics() displays one matrix
  • print_edge_prevalence() returns a message (not applicable)

K>1 Models

  • summary_multivar() shows common + subject-specific statistics
  • print_dynamics() displays matrices for all subjects
  • print_edge_prevalence() shows edge overlap across subjects

TVP Models

  • summary_multivar() shows per-period statistics
  • print_dynamics() displays time-varying matrices with period labels
  • print_edge_prevalence() shows prevalence separately for each period (if K>1)

Summary

The multivar summary functions provide:

✓ Comprehensive information about model structure and results ✓ Clear visualization of dynamics with readable formatting ✓ Cross-subject comparisons via edge prevalence ✓ Automatic detection of model types (TVP, K=1 vs K>1) ✓ Flexible customization for different display needs

These tools make it easy to explore, interpret, and communicate your multivar model results.
